Rolling back data (undo!)

It is possible to roll back an Envelope to effectively undo the loading of data into a Linkage Project. There could be a number of reasons that an operator would want to do this - for example:

  • The wrong data was loaded
  • The match or group configuration was not set correctly
  • The operator is testing the results of different configurations with sample datasets

A rollback undoes anything that was done as part of the last Envelope processing. For example, if the last envelope processed was the result of a delimited file that was uploaded, the rollback will undo any changes to the project database since that the file was loaded. All records added will be removed and any changes to existing groups will be undone.

info

A rollback can only be performed on the last Envelope for a Linkage Project.

An envelope can be rolled back through the JOBS tab of the web UI. On the ALL ENVELOPES sub-tab, select the Rollback icon in the Options column of the Envelope you wish to roll back. You will be requested to enter a reason for the roll-back – any reason can be entered here but this is more to confirm that you actually want to do this.

The last envelope submitted for each Linkage Project can be rolled back through the Jobs screen

Once an Envelope has been successfully rolled back, the status of the Envelope on the Jobs screen is updated to indicate this, and the previous Envelope for that project will now be available to roll back. Rollbacks can continue back to the very first envelope submitted for the project.

danger

There is no coming back from a rollback. It will delete any records that have been added since the last Envelope was created, including quality reviews!
