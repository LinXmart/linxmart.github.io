Loading data directly from a database

Loading data directly from a database requires some additional context, such as the intended Linkage Project and the Event Type. The Event Type also requires a configured Data Source that supports SQL Tables.

Loading data directly from a SQL table triggered from within a Linkage Project, and can be done in two ways:

  • Directly from the Linkages panel on a specific Linkage Project summary page
  • From the Event Type Data Sources page of a specific Linkage Project

Load new data for linkage​

On the summary page for a Linkage Project, there is a Linkages panel that will contain a list of all linkages that have been run in the context of the project.

Click on the Load New Data For Linkage button/link below this list.

You are presented with a form where you will need to choose:

  • the Event Type to classify the data
  • the Data Source that defines the format of the SQL table

Once each of these fields has been selected, click the Load button to confirm the data is ready to link. You will then be presented with a summary and confirmation form.

All SQL table data loads are presented internally as 'Envelopes'. At this point you have the ability to give the new Envelope a custom name or you can leave the default one that was generated for you. Click on the Load button here to trigger the direct access of data for linkage.

Load data from a specific data source​

The Event Type Data Sources page for a Linkage Project lists all Event Types and their associated Data Sources that are attached to the project.

For every Data Source that supports a SQL table, an load data icon is shown in the Options column. Click on this button and you will be presented with the 'Load Data' form, as you are when triggered from the Linkages panel. This time, however, the Event Type and Data Source are already populated with the required items.
