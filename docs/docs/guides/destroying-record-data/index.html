Destroying record-level data

LinXmart stores personally identifying information (PII) as part of the process of linking datasets together. However, sometimes it is required (by agreement) or desired to destroy the PII data after it has been used. LinXmart can clear the sensitive PII data stored for records without deleting the linkage itself.

There are several ways to trigger this:

  • Detaching an Event Type from a Linkage Project
  • Deleting an Event Type
  • Deleting a Data Provider
danger

These actions will irreversibly delete PII data within the system. They are considered high privilege actions, so if you are unable to see or access any of the described options, you may not have sufficient permissions. Please contact your LinXmart administrator.

Detaching an Event Type from a Linkage Project​

As well as deleting an Event Type completely from the system, an Event Type can be deleted from a specific Linkage Project. Effectively, this means the Event Type will be ‘detached’ from that project. It will still exist and be available in the system. To delete an Event Type from a Linkage Project, from the PROJECTS tab, select the project of interest and select ‘Event Types’ from the menu in the top right-hand corner. This brings up the Event Type Members screen (shown in Figure 41). From here Event Types can be deleted from a Linkage Project by selecting the icon in the Options column for the specific Event Type.

Figure 41: Event Types can be removed from a Linkage Project through the Event Type Members screen

Deleting an Event Type member from a Linkage Project will trigger an End-Date Event Type job.

This job will perform the following tasks:

  • Mark as End-Dated the Event Type’s appearance in the Linkage Project
  • Mark as End-Dated all records for this Event Type in the Linkage Project
  • Blank all personal identifiers
  • Mark as End-Dated all pairs and groups associated with these records

A grouping job will then be queued which will re-group all affected groups. Any further Linkage Requests for this Project that refer to the deleted Event Type will not be processed.

Deleting an Event Type​

An Event Type can be deleted from the Edit Data Provider Details screen. First select the owning Data Provider from the PROVIDERS tab. In the Data Provider Details section, select the Edit button in the top right-hand corner, bringing up the Edit Data Provider Details screen (Figure 40). From here, for the Event Type of interest, click the icon from the Options column to delete.

Figure 40: Event Types can be deleted through the Edit Data Provider Details screen

This will trigger an End-Date Event Type job.

The End-Date Event Type job will perform the following tasks:

  • Mark as end-dated all records for this Event Type that appear in any Linkage Project.
  • Blank all personal identifiers (all column values other than the Source Unique ID).
  • Mark as end-dated the Event Type itself.
  • Mark as end-dated all groups and pairs containing a record of this Event Type.

After this job is complete, a Group Pairs and Events, and an Execute Grouping Rules job for that Event Type will be queued for each relevant Project. This will regroup all groups and events affected by the above deletion. Any further Linkage Requests for this Event Type will not be processed.

Deleting a Data Provider​

A Data Provider can be deleted by selecting the option for the appropriate Data Provider listed on the PROVIDERS tab. This will trigger an End-Date Data Provider job.

The End Date Data Provider job will perform the following tasks:

  • Mark as End-Dated the Data Provider
  • Mark as End-Dated all Event Types belonging to the Data Provider
  • Blank all personal identifiers in all records belonging to the Data Provider in all Linkage Projects.
  • Mark as End-Dated all records belonging to the Data Provider in any Event Type in any Linkage Project, along with all pairs and groups containing those records.

After this job is complete, a Group Pairs and Events, and Execute Grouping Rules job will be queued for each project. This will regroup all groups and events affected by the above deletion.

Any further envelopes submitted for this Data Provider will not be processed.
